----- AI Results: ------
Generalized the dblock engine across three files:

- =updateDblock.py= :: Rewritten as a general dispatch engine. Handlers register under a
  signature string via =registerHandler(sig, fn)=. =expandAll(filePath)= does a single-pass
  =re.sub= over the file, parsing each =####+BEGIN: <signature> [:key value ...]= line into
  signature + keyword-arg dict, looking up the handler, and replacing the block body with
  the handler's return value. Unregistered signatures are left untouched. =targetDir= removed
  from public API — derived internally as =filePath.parent=.

- =dblock_particulars.py= :: Rewritten as a self-registering handler. Calls
  =updateDblock.registerHandler('b:ai:file/particulars', _handler)= at import time.
  Derives =activity= from the =AI-Activity.org= symlink and builds companion-docs list
  by scanning the directory. Accepts =args= dict for future parameterised dblocks.

- =py-dblock.cs= :: New CS executable with =updateDblocks= command. Takes one or more
  file paths as positional args (=argsMin=1 argsMax=9999=), calls =updateDblock.expandAll=
  on each. Imports =dblock_particulars= to self-register the handler.

Also: added =chmod +x *= to =.claude/settings.json=; added "Claude Permissions" section
to =mother/AI-WORKFLOW.org= directing Claude to always offer to persist new permissions
